# Product AI-APP Module Guidelines

产品页 AI-APP 界面模块统一按以下规则执行。适用于 AI 引流、AI 招商、AI 运营、AI 培训的所有大 Tab 和子 Tab。

## 模块固定规格

- 桌面端 `.product-stage` 高度固定为 `820px`。
- 不因左侧文案、子模块数量、右侧截图比例变化而修改外层模块高度。
- 同一个 AI-APP 内所有 Tab 的 `.product-stage` 高度必须一致。
- 同一套 AI-APP 模块的外层圆角、背景、左右栏宽度、底部 Tab 位置必须保持一致。
- 右侧截图、左侧说明栏都必须在固定模块内适配；如果内容放不下，优先压缩文案、减少子模块文字、调整内部间距，不改外框高度。
- 保持现有左右结构宽度比例，不为了图片或文案临时改左右栏宽度。
- 每次改动前先确认当前 Tab 的外层模块尺寸，改动后再对照已确认 Tab，确保没有高度跳变。

## 左侧说明栏

- 左侧说明栏不是长文本区，要做成“标题 + 一句话解释 + 图形化业务结构”的信息面板。
- 标题使用原产品模块标题字号规则：`.product-task-panel h3`，桌面端约 `38px`，不要随意缩小。
- 标题过长时用手动换行控制为 2-3 行，优先 3 行；不要让浏览器随机断句。
- 标题不加句号。
- 小标题 `product-kicker` 保持原样：大写英文、13px、900 字重、轻微字距。
- 标题下方说明文字控制在 2-3 行，解释该 Tab 的业务判断逻辑，不写泛泛的功能介绍。
- 不保留大面积空白；中部必须用图形化子模块填充。
- 默认不使用黑色 `业务输入` 大卡片。只有该 Tab 明确需要展示自然语言指令时才使用。
- 如果使用“业务输入”，必须做成标签框，并确保正文在目标行数内，不允许撑高模块。

## 左侧子模块结构

左侧内部优先采用下面的结构组合：

- 主判断卡：概括该 Tab 的核心判断对象，例如 `渠道 x 预算 x 到店 x ROI`。
- 维度网格：2 x 2 小卡，表达关键输入维度、状态、边界或规则。
- 流程线：3 段以内，例如 `数据接入 -> AI 判断 -> 生成动作`。
- 输出动作：3 个短标签，例如 `加码`、`暂停`、`审批`。

子模块文案规则：

- 每个小卡标题 2-4 字，说明 4-8 字。
- 避免完整长句，优先用业务名词和动作短语。
- 图形化元素要服务页面标题和右侧截图，不做纯装饰。
- 子模块之间留紧凑但清晰的间距，避免中间大空白。

## 当前 AI 引流 / 投放矩阵左侧标准

当前已确认的左侧结构：

- 标题：`先判断渠道 / 情况，再决定 / 投放动作`
- 标题为 3 行，使用原标题字号，不加句号。
- 说明：解释渠道表现、预算消耗、到店转化和 ROI 在同一张矩阵里完成判断。
- 中部结构：
  - 主判断卡：`矩阵判断`，`渠道 x 预算 x 到店 x ROI`
  - 四个判断卡：`渠道`、`预算`、`ROI`、`边界`
  - 流程线：`数据接入 -> AI 判断 -> 生成动作`
  - 输出动作：`加码`、`暂停`、`审批`
- 已去掉黑色 `业务输入` 模块。
- 外层模块高度保持 `820px`，与素材库 Tab 一致。

## 右侧截图固定规格

- 每张截图资源输出为 `1710 x 1040` PNG。
- 对应页面内显示区域为 `855 x 520`，即 2x DPR 截图。
- 不改产品页模块、外部卡片、浏览器框、文字区的宽高和布局。
- 只替换真实截图图片文件，不通过改页面高度/宽度来适配截图。
- 截图展示槽位的圆角、边距、底部子 Tab 与原模板保持不变；通过处理截图资源本身来匹配槽位。

## 右侧截图嵌入方式

- 先锁定产品页原有截图槽位尺寸，再处理截图素材。
- 不把截图撑满整个右侧内容区；必须保留原模板中截图与标题、左右边缘、底部子 Tab 之间的间距。
- 截图资源应直接填入原有圆角截图槽位，不在截图外再套新的框、阴影或背景。
- 如果真实截图比例不合适，重新裁切或缩放截图素材，不改产品页容器。
- 真实截图要清晰，优先使用 2x DPR 截图，避免在页面内二次放大导致模糊。

## 右侧截图范围

- 只截 AI-APP 左侧业务界面，不截右侧常驻 AI 对话框。
- 截图必须填满 `855 x 520` 比例，避免左右或上下白边。
- 截图内容应从真实业务白色内容卡片开始，不要截入页面左侧/顶部灰色背景。
- 图片自身可以保留真实界面的白色卡片和圆角，但不要额外再套一层截图框。
- 如果页面内容比例不匹配，调整截图裁切位置和滚动位置，不调整产品页容器。

## 右侧截图内容选择

- 每个大 Tab 下 4 张子 Tab 图，都使用对应子功能的真实业务界面局部。
- 截图内容要和下方子 Tab 标题一致，不要多个子 Tab 使用高度重复的区域。
- 优先选择信息密度高、标题清晰、业务动作明确的区域。
- 如果目标功能信息较少，可以截取同页相关的二级设置/明细区域，但必须围绕该子 Tab 的核心含义。

## AI 对话框类 Tab 截图

适用于网页端右侧常驻 AI 对话框，移动端对应为独立对话界面的 Tab。AI 引流、AI 招商、AI 运营、AI 培训中凡是“AI 指令/AI 中枢/AI 助手”类页面，都按这个逻辑处理。

- 截图对象是右侧 AI 对话区域，不是左侧业务工作台。
- 截图仍输出为 `1710 x 1040` PNG，最终填入产品页 `855 x 520` 截图槽位。
- 可以保留对话框顶部引擎状态栏、消息区和关键浮层，但不要截入浏览器外壳。
- 若云端右侧面板宽度略大于 `855`，先按真实右侧面板完整宽度截图，再缩放成 `1710 x 1040`，避免裁掉右侧按钮或历史侧栏。
- 最终处理图片时不得为了避免缩放而直接裁掉左右边缘；必须保留完整面板宽度，再缩放到 `1710 x 1040`。
- 四个子 Tab 的推荐截图状态：
  - `指令中心`：直接截初始 AI 指令中心/推荐指令界面。
  - `策略拆解`：点开顶部自主引擎 `详情`，截展开后的策略/工作流拆解页面。
  - `自动建议`：点击一个预设指令并运行，等待结果生成后截图运行结果界面。
  - `执行记录`：点击 `查看历史指令记录`，截弹出的历史指令侧边栏。
- 自动建议截图必须展示已生成的可执行结果，不截“AI 正在思考”的加载态。
- 历史记录截图必须显示侧边栏标题、至少一条历史指令卡片和状态，不截空历史。
- 如果 `指令中心` 首屏中间空白过大，先截取更长的右侧面板，再在图片素材里删除中间空白段，保留顶部中枢/引擎状态和下方推荐指令；压空白时要保留一段垂直缓冲，避免图标贴住顶部状态条。最终仍输出 `1710 x 1040`，不通过修改产品页容器适配。

## 配置中心类 Tab 截图

适用于 AI 引流、AI 招商、AI 运营、AI 培训的“配置中心 / 我的设置 / 系统设置”类 Tab。不同 AI-APP 选择的二级菜单可以不同，但截图方式和产品页呈现规则保持一致。

- 截图对象是左侧设置二级界面，不截右侧常驻 AI 对话框。
- 必须先进入具体二级设置页后再截图，不截设置列表总览页。
- 每个配置中心 Tab 选择 4 个最能体现该 AI-APP 设置特色的二级界面。
- 下方 4 个子 Tab 标签必须与所选二级界面一一对应，标签名可以按产品表达精简。
- 二级菜单选择依据：优先包含 `Skill`、`SOP`、`指标/预警`、`数据/系统接入` 这类能体现“方法变成可执行配置”的页面；如果某个 AI-APP 没有对应项，选择同等作用的配置页。
- 截图仍输出为 `1710 x 1040` PNG，最终填入产品页 `855 x 520` 截图槽位。
- 裁切时从二级界面的白色内容卡片开始，避免截入左侧/顶部灰色背景、设置列表外壳或右侧 AI 对话框。
- 如果二级页内容较长，优先截标题、核心配置卡、关键规则、开关、阈值和落点信息；不要只截空表单或底部保存按钮。
- 如果二级页首屏信息密度不足，可以滚动到该页最能表达配置价值的区域，但不能截半个关键模块。
- 左侧说明栏仍使用投放矩阵标准结构：标题 + 说明 + 主判断卡 + 2x2 维度卡 + 三段流程 + 三个输出动作。
- 外层 `.product-stage` 高度固定 `820px`，不因设置页截图或左侧文案改动模块高度。
