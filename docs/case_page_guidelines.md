# 案例页面设计准则

本文档以当前「粒子空间」和「智能机器人」案例为基准，用于后续案例的统一改造：美容医美连锁、高校继续教育学院。

## 1. 页面核心思路

案例页不是项目介绍页，也不是行业方案页。每个案例都必须回答同一个问题：

> 从这个案例里，能看清 AI 增长解决方案先改造了哪些关键环节，以及这些环节如何变成团队可复用能力。

统一表达逻辑：

1. 先定位最值得验证的增长动作。
2. 再拆成 `判断 / 执行 / 复盘` 三个 AI 改造环节。
3. 最后落到可复用的结果：获客、成交、资产、复购、团队方法或渠道能力。

禁止把左侧模块写成单纯的项目背景、产品介绍、设备介绍或商业计划摘要。

每个 Tab 的左右模块都必须服务该案例 Hero 主标题和副标题的核心判断。页面是在展示「AI 增长解决方案如何帮助客户把关键增长环节改造为可理解、可验证、可复用的动作」，不是展示客户行业、产品或设备本身。行业和产品信息只能作为场景素材与证据，不能成为模块主旨。

验收口径：

- 如果模块读起来像智能硬件、护肤、医美、视光或教育行业介绍，而看不出 AI 增长方案的判断、执行、复盘作用，则必须重写。
- 首个 Tab 的真实场景图负责建立行业与产品直观认知；旁边图形模块必须回答“AI 如何把这个场景转成增长动作”。
- 文案优先使用 `AI 增长方案 / 增长动作 / 判断 / 执行 / 复盘 / 证据 / 可复制能力` 的逻辑，不用空泛的客户价值介绍替代增长机制。

## 2. 案例区主标题与副标题

当前粒子空间案例区标题作为风格基准：

- 标签行：`粒子空间 / 精准护肤平台` 与 `AI 招商 · AI 引流 · AI 运营` 同行并列居中。
- 主标题：`把精准护肤体验变成 / 可解释、可建档、可复购的增长闭环`
- 副标题：`业务增长的关键不是多卖一款护肤品，而是降低每一次护肤决策的风险。 / 从测肤解释、Skin ID 建档到复购提醒，把一次体验沉淀为长期客户资产。`

后续案例标题规则：

- 主标题必须两行以内，居中显示，沿用产品页/当前案例页的大号展示字体风格。
- 主标题不加引号，不写空泛口号，要突出“AI 增长解决方案如何重构该行业的关键增长闭环”。
- 副标题放在主标题下方，最多两行，负责解释“为什么这是增长问题，而不只是产品/服务问题”。
- 标题避免只讲行业价值，要讲 AI 介入后的增长机制。

相关样式基准：

- `.case-heading-centered`：居中，最大宽度 `1180px`。
- `.case-label-row`：居中并列，间距 `10px`，底部间距 `18px`。
- `.cases-page .section-title`：`clamp(42px, 4.8vw, 70px)`。
- `.section-lede`：`clamp(18px, 1.45vw, 22px)`，行高 `1.48`。
- `.case-heading-lede`：最大宽度 `860px`，顶部间距 `22px`。

## 3. 每个案例的 Tab 结构

每个案例固定 4 个 Tab。Tab 不是功能罗列，而是增长链路拆解。

推荐结构：

1. `入口 / 场景 / 首个增长动作`：真实场景图 + AI 增长动作图。
2. `信任 / 判断 / 解释`：AI 如何把经验判断变成可解释内容。
3. `资产 / 建档 / 回写`：AI 如何把一次动作沉淀成可复用数据、档案或规则。
4. `复购 / 招商 / 复制 / 训练`：AI 如何把已验证动作变成团队可复制能力。

Tab 命名应短，优先使用业务动作词，例如：`补鲜站入口`、`AI 测肤报告`、`Skin ID 资产`、`复购与招商`。

同一案例内四个 Tab 的图形表达必须有足够差异化：

- 不能只替换文案套用同一个左侧模块或右侧图片模板。
- 每个 Tab 的左侧模块和右侧图都必须紧贴该 Tab 自身内容：入口、报告、资产、复购/招商分别使用不同的信息结构。
- 允许共享整体视觉语言、色彩和字体，但不允许复用同一种核心图形，例如连续多个 Tab 都使用中心环形、左右侧栏、四角卡片或同一种动作映射。
- 调整某个 Tab 时，参考图只能参考表达形式；不得把其他 Tab 或参考图中的业务内容、步骤和图形结构机械套入当前 Tab。
- 验收时要同时看“内容是否贴合”和“图形是否区分”。如果内容正确但图形像前一个 Tab，也视为未完成。

## 4. 固定高度与布局约束

桌面端案例 Tab 舞台必须固定高度，不能被左侧内容撑高。

当前基准：

- `.particle-growth-lab .case-tab-stage`：`height: 690px; min-height: 690px;`
- 左右布局：`grid-template-columns: minmax(300px, 0.72fr) minmax(0, 1.28fr)`
- 左右间距：`28px`
- 左卡和右图卡高度必须一致。
- 内容多时先调整文案、字体、行高、结构密度，不允许拉高整个模块。

移动端可切换为单列与自适应高度，但不能横向溢出。

验收口径：

- 桌面端左卡 `scrollHeight <= clientHeight`。
- 左卡内容整体垂直居中，上下边距尽量一致。
- 不允许底部结果区被裁切。

## 5. 左侧模块：AI 增长图谱

左侧模块必须是咨询风格的图形化框架，不是大段文本框。

当前粒子空间首个 Tab 的核心结构：

1. 顶部标签：`AI growth map`
2. 标题：`首次进站，验证可复用增长链路`
3. 动作映射：`9.9 新客体验包 → AI → Skin ID`
4. 改造环节：`01 判断 / 02 执行 / 03 复盘`
5. 关键结果：`获客 / 成交 / 资产`

左侧标题规则：

- 标题必须是业务判断句，不写泛泛口号。
- 标题末尾不加句号。后续所有案例 Tab 左侧标题统一不使用句号收尾。
- 同一案例内不同 Tab 的标题字号必须一致。当前粒子空间基准为 `clamp(30px, 2.35vw, 35px)`，行高 `1.08`。
- 标题可以自然换行，但不要为了逗号强制换行。由容器宽度自然折行。

后续案例必须套用同一抽象：

- `增长动作`：该案例最值得先验证的一步。
- `AI`：AI 承接解释、判断、提醒、生成、跟进、复盘或训练。
- `复用资产`：沉淀成档案、规则、脚本、SOP、渠道证据、客户数据或团队 Skill。
- `判断`：AI 先帮助团队判断什么最值得做。
- `执行`：AI 把动作变成标准化内容、流程或工具。
- `复盘`：AI 把结果回写为下一轮动作依据。
- `关键结果`：用 3 个短词收束，避免长句。

当前字体与尺寸基准：

- 左卡背景：深黑 `#101010`。
- 左卡 padding：`26px 32px`。
- 内容整体：`align-content: center`。
- 左卡标题：`clamp(30px, 2.35vw, 35px)`，行高 `1.08`。
- 顶部标签：`12px`，大写英文，胶囊样式。
- 动作节点标题：`17px`，`white-space: nowrap`，确保如 `9.9 新客体验包` 单行显示。
- 三步标题：一行显示，例如 `01 判断`，不要拆成两行。
- 三步标题字号：`17px`。
- 三步说明：`12px`。
- 结果标题：`12px`。
- 结果主词：`16px`。
- 结果说明：`12px`。

图标规则：

- 三步环节如使用图标，必须与含义相关。
- `判断` 可用放大镜、筛选、靶心类 SVG。
- `执行` 可用路径箭头、流程箭头、工具类 SVG。
- `复盘` 可用循环箭头、回写、刷新类 SVG。
- 不使用无意义的圆点、装饰圈、抽象形状占位。

AI 测肤报告类 Tab 的左侧图形规则：

- 可采用“四角问题卡 + 中心判断卡 + 底部三条动作卡”的咨询图形布局。
- 中心判断卡当前基准高度为 `120px`，内容必须纵向居中，内部元素之间要留出清晰间距。
- 中心判断卡与上下卡片之间的间距必须一致，不允许压住上下模块。
- 底部三条动作卡使用统一文本框样式，三条高度一致。说明文字优先压缩为一行，避免撑高模块。
- 底部动作卡如 `01 解释风险 / 02 承接首购 / 03 回写档案`，数字、标题、说明保持同一行栅格，不要出现某一条换行导致高度不一致。

Skin ID 资产类 Tab 的左侧图形规则：

- 左侧模块必须继续体现案例 Hero 的核心思想：AI 不是展示档案，而是把一次关键动作沉淀成后续可复用的增长依据。
- 文案必须贴近业务动作，避免使用用户难以理解的抽象词。不要写 `体验信号`、`回写` 这类内部表达；优先写 `一次测肤`、`客户账户`、`复购依据`、`复购周期`、`周期提醒`、`后续跟进依据`。
- 当前粒子空间 Skin ID 左侧基准标题：`把一次测肤，沉淀为复购依据`。标题末尾不加句号。
- 表达形式要与前两个 Tab 明显不同。不要复用 `补鲜站入口` 的动作映射，也不要复用 `AI 测肤报告` 的四角解释图。
- 推荐采用“客户账户 / 资产账本 + 复购提醒矩阵 + 关键结果”的结构：
  - 顶部说明：Skin ID 不是只建档，而是把肤况、组合、反馈和复购周期变成后续跟进依据。
  - 中部账户：`测肤结果 → 组合方案 → 护理反馈 → 复购周期`。
  - 字段卡：`肤况 / 组合 / 反馈 / 周期`。
  - 动作卡：`01 建档 / 02 提醒 / 03 升级`。
  - 关键结果：`资产 / 提醒 / 复购`。
- 模块仍必须固定在 `690px` 高度内，内容整体垂直居中；若文案换行导致溢出，优先压缩文案、间距和局部字号，不允许撑高模块。

复购与招商类 Tab 的左侧图形规则：

- 左侧模块必须表达“复购能力如何成为招商可复制能力”，不能写成普通总部能力介绍。
- 文案应来自复购与招商本身：`入私域率`、`30/60/90 日复购`、`总部托管`、`SOP 监管`、`培训题库`、`分成结算`、`点位模型`。
- 当前粒子空间左侧基准标题：`把复购能力，变成可招商的运营系统`。标题末尾不加句号。
- 表达形式必须区别于前三个 Tab。当前基准采用“复购证据 → 总部托管 → 招商复制”的阶梯式证明链，而不是动作映射、四角解释图、客户账户卡或生命周期环形。
- 推荐结构：
  - 顶部说明：招商卖的不是一个点位，而是总部托管的获客、复购、内容和训练能力。
  - 阶梯证明链：`01 复购证据 / 02 总部托管 / 03 招商复制`。
  - 转化关系：`复购能力 → 招商口径`。
  - 关键结果：`复购 / 托管 / 复制`。
  - 模块仍必须固定在 `690px` 高度内，内容整体垂直居中且不得溢出。

## 6. 右侧图片模块

右侧必须承载真实或拟真的案例场景，而不是纯抽象 UI。

当前粒子空间首个 Tab 基准：

- 图片文件：`assets/images/cases/lizi-space/lizi-entry-station-tall.png`
- 图片用途：基于真实补鲜站场景，通过 `gpt-image-2` 扩展高度以适配右侧图片框。
- 图片处理：不加模糊滤镜，不压缩成错误比例，不让主体被裁掉。
- 图片渲染：`object-fit: cover`，铺满图片区。
- 右图图片区：白底，圆角，图片框内不再叠加旧的抽象伪元素。

右图标题栏规则：

- 顶部标签如 `补鲜站入口图`，字号 `14px`。
- 标签用浅暖白背景、浅粉边框、暖棕文字，避免黑色硬标签。
- 分割线使用浅暖粉灰，不使用深灰重线。
- 右上三点降低对比度，作为轻量窗口暗示。

右下/左下说明层规则：

- 保留图片内部的信息标识，但必须轻量，不抢主体。
- 当前基准：
  - `低门槛体验入口`
  - `2-5㎡ 补鲜站`
  - `9.9 新客体验 · 测肤入口 · Skin ID 建档`
- 说明层使用暖粉半透明底、浅边框和轻阴影。
- 主文字使用展示字体，`clamp(29px, 2.25vw, 40px)`。
- 辅助文字 `12-13px`。

后续案例图片应按相同方法生成或处理：

1. 找真实场景或明确业务场景图片。
2. 使用图像模型扩展或重绘为右图框适配比例。
3. 保持案例行业真实感，不做纯装饰图。
4. 根据案例色彩轻微调整说明层，但结构一致。

复杂 UI / 报告类右图规则：

- 复杂 UI 图优先使用 `gpt-image-2` 基于已确认图片进行编辑，不使用 HTML/CSS 重新搭建最终图片。
- 生成前必须锁定输出比例。当前粒子空间右图基准为 `1600x1280`，约 `5:4`，用于匹配右侧图片区。
- 页面渲染时不依赖 `transform: scale(...)` 修正白边或裁切。图片应在生成阶段适配比例，页面中保持 `transform: none`。
- 图片内容要“编辑原图”，不要重画成另一套仪表盘。提示词必须明确：保留原图结构、卡片、标签、图标和内容层级，只调整布局、字号和裁切。
- 当右侧内容被裁切时，优先压缩或重排左侧身份区，而不是缩小主内容。例如：删除图内冗余大标题；`Skin ID` 与编号分两行；流程从横向改为纵向排列，为右侧报告区释放空间。
- 图片内字体必须统一，避免不同字体混用。说明小字、标签文字、雷达图标签、底部导航文字要尽量放大，但不得溢出卡片。
- 文字密集图必须检查网站展示尺寸下的可读性，不只看原始大图。若小字在页面内不可读，需要重新生成或放大字号。
- 右边缘、底部导航和主要卡片必须完整显示，不允许被图片区裁掉。

Skin ID 资产类右图规则：

- 当内容主要是客户生命周期价值、账户资产、复购周期和编码体系时，可以使用 HTML/CSS 作图，不必调用图像模型。
- 右图应采用生命周期表达，而不是普通卡片并列。当前粒子空间基准是：中心 `Skin ID` 资产核，外圈五步生命周期节点，左右两侧分别承载 `User Skin ID` 与 `生命周期价值`，底部承载 `User Skin ID / Particle ID / Routine Code` 编码资产。
- 生命周期节点使用现有业务内容：`测肤建档 / 组合首购 / 使用反馈 / 场景触发 / 复购升级`。不要因为参考图里有其他词而替换当前案例内容。
- 用户提供的截图只作为“形式和图形化表达”的参考。严禁把参考图里的标题、步骤、文案、图标或业务内容直接套到当前图里。
- 外圈必须保持正圆，不要因容器比例变成椭圆；节点位置需要围绕环形自然分布，底部节点的数字应落在圆环上方或贴近圆环，不要漂在环内。
- 左右两侧信息栏可以适当收窄，把视觉重心留给中间生命周期图；中心模块宽度应大于左右侧栏。
- 中心核只保留必要信息。当前基准为 `Skin ID` + `可追踪 · 可复购 · 可分析`，不显示 `Digital asset base` 等占空间且信息量弱的小字。
- 右图所有修改必须先保留现有内容，再调整表达方式；除非用户明确要求，不得重写内容。

复购与招商类右图规则：

- 右图可以使用 HTML/CSS 作图，但不能沿用 Skin ID 资产 Tab 的中心环形生命周期模板。
- 图形核心应来自案例文件中的“加盟投资 + 总部直管 + AI 托管运营”与“私域复购 + 会员运营 + 加盟商管理”。
- 当前粒子空间右图基准采用“总部 AI 托管作战板 + 双线流程”：
  - 顶部：`总部 AI 托管中台`，包含 `策略分发 / 内容投放 / SOP 监管 / 数据复盘`。
  - 中部双线：`私域复购链` 与 `直管招商链`。
  - 复购链：`Skin ID 账户 / 周期提醒 / 会员菜单`。
  - 招商链：`加盟投资 / 总部直管 / AI 托管运营`。
  - 底部验证指标：`入私域率 / 30/60/90 日复购 / 回本周期 / 点位收益`。
- 右图要像运营作战板或流程看板，不要出现中心圆核、环形轨道、生命周期外圈等 Skin ID 资产图语言。
- 右图必须解释招商为什么能成立：先有复购数据与私域能力，再由总部把内容、SOP、训练、数据和分成托管给点位。
- 所有内容必须紧贴复购与招商主题；不允许为了形式好看引入与本 Tab 无关的图形或文案。

## 7. 视觉风格

统一风格关键词：

- 咨询式
- 图形化
- 真实业务场景
- 高密度但不拥挤
- 左侧深色增长框架，右侧浅色场景图
- 少解释、多结构

色彩规则：

- 官网整体仍是蔚为 AI 增长解决方案，不要变成某个行业品牌站。
- 案例可以使用行业色作为局部点缀。
- 粒子空间可用浅粉、暖白、香槟金，但必须克制。
- 左侧深色卡使用暖金/暖橙作为信息点缀。

组件规则：

- 不使用嵌套卡片堆砌来替代结构。
- 不使用无意义装饰图标。
- 每个图形元素都必须表达业务含义：动作、AI、资产、判断、执行、复盘、结果。
- 文字必须适配容器，不能溢出或靠裁切隐藏。

## 8. 智能机器人案例沉淀准则

智能机器人案例的核心不是介绍巡逻机器人功能，而是说明 AI 增长方案如何把复杂硬件卖点转译为客户可理解、可试用、可培训、可持续跟进的增长动作。

Hero 区规则：

- 主标题和副标题沿用粒子空间案例的居中、大号展示字体、两行以内的格式。
- 如果用户明确要求保留原文，只改格式，不重写标题内容。
- 主标题可按用户指定的标点自然分行；副标题可按分号分为两行。没有明确要求时，不要强行按逗号换行。
- 智能机器人案例基准副标题逻辑：智能硬件成交关键在场景价值、试用证据和渠道培训；业务增长难点不是产品不强，而是客户不容易理解为什么现在要买。

四个 Tab 的内容与表达基准：

1. `场景成交`
   - 左侧图片使用真实或官网场景素材，输出“产品 + 场景”的一张主视觉图，让用户快速理解行业、产品和应用场景。
   - 图片说明卡只保留关键信息，不出现客户名称和 Logo；卡片应轻量、小尺寸，不遮挡产品主体。
   - 右侧模块必须回答“AI 如何把硬件能力改写成客户价值”，当前基准为深色 `AI SCENE VALUE MAP`：产品参数 → AI 增长翻译 → 成交语言。
   - 图形结构可以用左右映射，但只适合首个场景成交 Tab，不应在后续 Tab 重复。

2. `试用报告`
   - 左侧图片适合用 HTML/CSS 作图，核心是 AI 试用报告如何把试用数据转成订阅购买依据。
   - 报告图不加外层框套框；图片内部应多用图表表达，少用纯文字。
   - 当前基准内容包括：巡逻覆盖图、告警事件分析、整体评价雷达图、试用转订阅流程。
   - 右侧模块不能复用场景成交的参数映射结构。当前基准为“订阅决策备忘录”：客户购买问题 → AI 判断 → 增长动作。
   - 标题不要使用“购买证据”等生硬表述，应表达试用报告如何推动订阅决策。

3. `渠道能力`
   - 左侧图片适合用 HTML/CSS 作图，核心是总部如何把场景价值、试用证据和部署边界变成渠道伙伴可学习、可演练、可复用的能力包。
   - 图片可以采用“总部方法库 + 训练路径 + 能力输出”的表达，但标题和说明必须足够清晰，字号不能过小。
   - 文本框标题前可以使用有意义的 SVG 图标，图标必须服务阅读，不做无信息量装饰。
   - 右侧模块必须完全区别于试用报告 Tab。当前基准为“AI 渠道培训教室 / Skill OS”：总部方法 → 伙伴席位训练 → AI 培训输出。
   - 右侧标题基准：`利用AI培训将总部方法赋能给渠道伙伴`。
   - 输出标签使用短词：`渠道话术 / 试用报告 / 部署问答`，避免过长导致断行混乱。

4. `线索跟进`
   - 左侧图片适合用 HTML/CSS 作图，核心是 AI 如何帮助客户经营，而不是普通 CRM 或产品清单。
   - 当前基准为“客户经营图”：客户分层矩阵 + AI 下一步动作 + 经营闭环。
   - 客户分层矩阵的四个文本框样式必须一致；轴线文字如 `场景匹配` 可竖排，但要与卡片留出足够间距。
   - 说明文字需统一断句和字号，保证四个象限看起来整齐。
   - 右侧模块必须完全区别于前三个 Tab。当前基准为“AI 客户经营节奏盘”：经营信号 → AI 优先级判断 → D0/D3/D7/D14 跟进节奏。
   - 右侧标题应体现 AI 增长方案对线索跟进的提升，例如 `用AI提升线索跟进效率`，不要只写普通客户转化口号。
   - 中间圆盘、轨道点和右侧文本框必须避免重叠；如果发生遮挡，优先缩小核心字号、调整圆盘尺寸和卡片位置。

智能机器人案例对后续案例的通用提醒：

- 每个 Tab 都必须回到案例 Hero 的主副标题核心思想：AI 增长方案如何把难理解、难验证、难复制的环节变成可执行动作。
- 同一案例的四个 Tab 不能都做成 `3 个步骤 + 3 个结果`，也不能只是横向改纵向。若用户要求“完全不同”，必须从信息结构和思路上重做。
- 图片和模块可以使用 HTML/CSS 或图像模型，选择依据是内容类型：真实场景优先图像模型，报告/矩阵/训练/经营系统优先 HTML/CSS。
- 无论采用哪种方式，都必须固定在 `690px` 桌面高度内。内容溢出时优先改文案、字号、行高、间距和布局，不允许撑高模块。
- 参考图只参考表达方式，不替换当前 Tab 的业务内容。

## 9. 后续案例改造方法

为每个案例先完成以下提炼，再动页面：

1. 从案例材料中提炼“最值得验证的增长动作”。
2. 定义该动作之前的关键判断。
3. 定义 AI 如何把执行动作标准化。
4. 定义执行结果如何复盘和回写。
5. 定义沉淀成什么可复用资产。
6. 定义 3 个关键结果词。
7. 生成或处理右侧真实场景图。
8. 按 `690px` 固定高度压缩内容。

输出模板：

```text
案例名称：
Tab 名称：

AI 承接：
复用资产：
01 判断：
02 执行：
03 复盘：
关键结果 1：
关键结果 2：
关键结果 3：
右侧场景图：
图片说明层：
```

完成后再进入 HTML/CSS 实现。
